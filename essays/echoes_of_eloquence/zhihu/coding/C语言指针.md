## 一、指针基本介绍

要理解指针，首先需要理解不同的数据类型或者不同的变量在计算机的内存中是如何存储的。

![ ](https://pic4.zhimg.com/v2-60ddba27a93db6a445fbf6e36e85befb_b.jpg)

图1.1

假设图1.1表示计算机内存，左侧的数字代表地址，每一个地址都可以存储1byte的数据。(1byte=8bit=8位二进制)

当在程序中声明一个变量时，计算机会为这个变量分配一定的内存空间，具体需要多少内存空间，取决于变量的数据类型以及编译器。

![ ](https://pic4.zhimg.com/v2-ea32b36d046a0d34b4202de9c76cd857_b.jpg)

图1.2

int类型变量会占据4个Byte，char类型变量会占据1个Byte，float类型变量会占据4个Byte。

在图1.2中，执行"int a"：计算机会为整型变量a分配4byte的内存，例子中假设变量a的内存地址范围是从204到207。

"char c"：计算机会找空闲的内存空间，为字符型变量c分配1byte内存，假设变量c的内存地址是209。

"a=5": 计算机会去查找a，并找到它的起始地址204，写入5这个值，实际上5这个值是以二进制的形式写入的。

![ ](https://pic2.zhimg.com/v2-78f1f8daa8406505cd1d8b70b1f05e51_b.jpg)

图1.3

"a++"：执行变量a自增(即a=a+1)时，计算机去查a的地址，然后修改这个地址存放的值，所以地址204-207现在就存放着6这个值。

那么能否在程序中知道变量的地址呢或者能不能通过这些内存地址进行操作呢，答案是可以的。

指针是一个变量，它存放着另外一个变量的地址

![ ](https://pic2.zhimg.com/v2-f0b538ae46449ec8b600d6c58acd5c95_b.jpg)

图1.4

指针是一个变量，它存放着另外一个变量的地址。可以通过指针知道变量的地址或者通过指针对地址进行操作。

图1.4中，假设有个4byte的一块内存，从地址204开始存放变量a。现在有一个指向整型变量的指针变量p，它的地址为64，也需要占据4byte，这个变量p用来存放变量a的地址(204)，然后就可以通过指针p去访问a，也可以修改p使其指向其他的整型变量。

假设有另外一个整型变量b，它的地址为208，值为10，如果将p的值从204改为208，那么p就会指向b。

int a: 声明一个整型变量a。假设a在计算机中的内存地址为204-207

int *p：声明一个指向整型变量的指针p。即将一个星号(*)放在变量名前面。现在可以说p是一个可以存放整型变量的地址的变量。假设p在计算机中的内存地址为64-67

p=&a：在p中存放a的地址。即将一个&放在变量前面，p就得到了a的地址。"&"表示取地址。

![ ](https://pic2.zhimg.com/v2-31fd2c3a9d8fbaffa796e79990383b95_b.jpg)

图1.5

a=5：初始化变量a，那么地址204-207存储的值就是5。

print p：打印p的值，输出会是204，即输出了a的地址。

print &a：打印&a，输出仍然是204，即输出了a的地址。&a表示a的地址。

print &p：打印&p，输出是64。p是一个指针变量，同样会存在内存中，所以&p，将会得到p的地址，所以输出64.

print *p：输出为5。将星号(*)放在**指针变量**的前面，那么会得到指针所指向地址的值，也就是该地址存储的值，这个概念称为解引用。p指向地址204，地址204存储的值为5，所以输出5。

![ ](https://pic3.zhimg.com/v2-477921c3e5323bc1094cff311cb88b3a_b.jpg)

图1.6

*p=8：可以通过指针修改特定位置的值。*p表示p指向地址的值，那么现在p所指向的值就改成了8。此时再执行print a，输出将是8。

### 本章总结

声明了一个指针变量时(int *p)：p的值表示p的地址，因此p表示地址。

而*p是一个值，这个值就是p指向的那个地址里面存储的值。

定义一个指针变量时，把星号(*)放在变量名前。例如int *p。

如果把&放在变量名前，会得到它的地址。例如print &a。

如果对一个指针变量不使用星号(*)，那么只是对指针变量的地址进行操作。例如print p。

当把星号(*)放在指针变量前面，是在操作指针所指向的那个地址的值。例如*p=8。

## 二、指针代码示例

![ ](https://pic2.zhimg.com/v2-35fe2902ca9378d0f64e38935d476275_b.jpg)

图2.1

图2.1中，为了存储整型变量的地址，需要一个指向整型的指针。即int *p。

为了存储字符型变量的地址，需要一个指向字符型的指针。即char *p0。

为了存储双精度浮点型变量的地址，需要一个指向双精度浮点型的指针。即double *p1。

同样，也可以用指针指向用户自定义的结构体。

p=&a：为了获取变量a的地址。即p=204，204为变量a在内存中的首地址。这样p就指向了a。

```c
//代码2_1.c
#include <stdio.h>

int main() {
 int a;
 int *p;
 printf("%x\n", p);

 return 0;
}
```

![ ](https://pic4.zhimg.com/v2-ae49d8aca980b84a5fa8e88ad6f9545f_b.jpg)

运行结果

2_1.c中并没有对指针变量p进行初始化，所以会弹出警告，部分编译器可能会弹出error。一定要在实际写代码的过程中避免这种情况，在定义了一个指针变量时，一定要先对其进行初始化。

```text
//代码2_2.c
#include <stdio.h>

int main() {
 int a;
 int *p;
 p = &a;      //&a -> 变量a的地址
 printf("%#x\n", p);    //以16进制格式打印p
 printf("%d\n", *p);    //*p -> p指向的地址里面存储的值

 return 0;
}
```

![ ](https://pic3.zhimg.com/v2-c6d75bca9f44de6381adeac502134ec6_b.jpg)

运行结果

2_2.c中初始化了指针变量p，使其指向整型变量a。代码中并未对变量a进行初始化，此时打印*p后，输出是0。这取决于编译器，有些编译器打印出来的*p可能是随机值。

```text
//代码2_3.c
#include <stdio.h>

int main() {
 int a;
 int *p;
 p = &a;       //&a -> 变量a的地址
 a = 11;
 printf("%#x\n", p);    //以16进制格式打印p
 printf("%d\n", *p);    //*p -> p指向的地址里面存储的值
 printf("%#x\n", &a);   //以16进制格式打印a的地址

 return 0;
}
```

![ ](https://pic3.zhimg.com/v2-15f5841c2e3b25d9d2ec45ba2a7815da_b.jpg)

运行结果

2_3.c中初始化变量a，a=11。因此print *p结果为11。通过p=&a，将a的地址赋给了p，因此print p和print &a的输出结果一致。

```text
//代码2_4.c
#include <stdio.h>

int main() {
 int a;
 int *p;
 p = &a;       //&a -> 变量a的地址
 a = 10;
 printf("a=%d.\n", a);
 *p = 12;
 printf("a=%d.\n", a);


 return 0;
}
```

![ ](https://pic3.zhimg.com/v2-020c8251de1fb021e7cb741e7ed40cf6_b.jpg)

运行结果

2_4.c中在第一次打印a的值时，此时输出为10。然后通过*p=12，改写了p所指向的地址中存储的值，因此再次打印a，输出为12。

```text
//代码2_5.c
#include <stdio.h>

int main() {
 int a;
 int *p;
 p = &a;       //&a -> 变量a的地址
 a = 10;
 printf("Address of p is %#x.\n", p);
 printf("Value at p is %d.\n", *p);
 int b = 20;
 *p = b;       //是否会改变p所指向的地址呢
 printf("Address of p is %#x.\n", p);
 printf("Value at p is %d.\n", *p);

 return 0;
}
```

![ ](https://pic2.zhimg.com/v2-71b7ea34cccb979f9e116aa3e4f64dd9_b.jpg)

运行结果

2_5.c中新定义了一个整型变量b，然后*p=b，是否会将b的地址赋给p呢，并不会。但是执行*p=b后会改写p指向的地址所存储的值，p仍然指向整型变量a，地址并未改变。

```text
//代码2_6.c
#include <stdio.h>

int main() {
 int a;
 int *p;
 p = &a;       //&a -> 变量a的地址
 a = 10;
 printf("Address p is %#x\n", p);
 printf("size of integar is %d bytes.\n", sizeof(int));
 printf("Address p+1 is %#x\n", p + 1);

 return 0;
}
```

![ ](https://pic1.zhimg.com/v2-c9b518907f738b4b9485594d7d91d714_b.jpg)

运行结果

2_6.c中分别打印了p和p+1，但为什么p+1之后是0x61fe18，而不是0x61fe15呢。因为p是一个指向整型类型的指针，对它进行+1操作，会得到下一个整型数据的地址。一个整型变量的大小是4byte，因此为了得到下一个整型变量的地址，我们会跳过4byte，所以p+1会增加4个字节(4byte)。

同理，如果是p+2，那么地址会+8，输出结果将是0x61fe1c，即0x61fe14+8。可以更改数据类型为char,double等观察p+1后的结果。

```text
//代码2_7.c
#include <stdio.h>

int main() {
 int a;
 int *p;
 p = &a;       //&a -> 变量a的地址
 a = 10;
 printf("Address p is %#x\n", p);
 printf("Value at p is %d.\n", *p);
 printf("size of integar is %d bytes.\n", sizeof(int));
 printf("Address p+1 is %#x\n", p + 1);
 printf("Value at p+1 is %d.\n", *(p + 1));

 return 0;
}
```

![ ](https://pic2.zhimg.com/v2-2297bdee55e3f182aa77ea606c0cea5d_b.jpg)

运行结果

2_7.c尝试着打印了*(p+1)，但是输出结果其实是一个随机值，因为并没有对该地址分配一个整型变量。这是使用指针算术运算的时候的一个危险的地方，利用指针，我们可以到达任何地址，但有时候，这种操作会引起不期望的行为。

### 本章总结

1、声明了一个指针变量时，必须要对其初始化。

2、对指针变量进行算术运算时，并不只是简单的算术运算。p+n的实际含义为 p + n*sizeof(data type)，其中p为指针变量，n为正整数。sizeof(data type)表示每种数据类型所占字节，例如int占据4byte，char占据1byte，double占据8byte......

3、通过指针去操作那些已经被分配变量的地址，不要操作那些未被分配变量的地址。

## 三、指针的类型，算术运算，void指针

指针是强类型的，这就意味着需要一个特定类型的指针变量来存放特定类型变量的地址。对于int*来说就需要一个指向整型类型的指针，来存放整型数据的指针。指向字符类型的指针来存放字符型数据的地址。

Q:为什么指针需要强类型，指针不就是一个存放变量的地址吗？为什么不用一个通用的类型存储所有类型变量的地址呢 ？

A:指针不仅用来存储变量地址，还需要用来解引用地址的内容，这样就可以访问和修改那些地址中的值了。

![ ](https://pic4.zhimg.com/v2-200b822f10f88c58f370cc702a657a23_b.jpg)

图3.1

比如有一个整型变量a，它的值是1025，变量a的内存布局如图3.1所示。每一个方框代表一个地址，每个地址可以存储1byte（即8bit）的数据。假设从右往左开始，分别是byte0, byte1, byte2, byte3，每个字节在内存中都是可以寻址的。这4个字节的地址必须是连续的，假设byte0的地址是200，byte1的地址201，然后是202和203。当整数在内存中表示时，最左边的bit表示符号位，用于表明这个数是正数还是负数，剩下的31个bit用来存储值。

因此1025在内存中的二进制表示方法就是**0000 0000** 0000 0000 **0000 0100** 0000 0001：最左边的0表示该整数为整数，剩下的31个bit用来表示1025：1*2^10+1*2^0=1025。

![ ](https://pic3.zhimg.com/v2-880d59656ca93bffb5f8622dc6c5f51e_b.jpg)

图3.2

图3.2中，声明一个指针p指向整型变量(int *p)，然后利用取地址符号(&)把变量a的地址存在指针p中(p=&a)，此时打印p的值，输出将是200，也就是byte0的地址，也就是说整型变量的起始地址是200。如果想要知道地址的内容，就使用*p去解引用那个地址，然后机器看到后，p是一个指向整型的指针因此需要看4个字节(byte)，那么就从地址200开始，机器知道怎样提取一个整型类型的值，所以机器从这4个字节中提取出了1025这个值。

如果p是一个字符类型的指针，那么在解引用的时候机器只会看1个字节(byte)，因为字符变量只有1个字节。

如果p是一个指向浮点类型的指针，那么在解引用的时候机器会看4个字节(byte)，但是浮点数的4字节表示和整型数的表示是不一样的。

```c
//3_1.c
#include <stdio.h>

int main() {
 //1025 = 00000000 00000000 00000100 00000001
 int a = 1025;
 int *p;
 p = &a;
 printf("size of integar is %d.\n", sizeof(int));        //通过sizeof()得到int所占字节
 printf("Address = %p, value = %d.\n", p, *p);
 char *p0;
 p0 = (char *)p;      //强制类型转换，因为p指向int类型，p0指向char类型
 printf("size of char is %d.\n", sizeof(char));
 printf("Address = %p, value = %d.\n", p0, *p0); 
}
```

![ ](https://pic2.zhimg.com/v2-af7914ef918fafeacf3a87e0fc996445_b.jpg)

运行结果

3_1.c中声明了一个字符类型的指针p0，然后将p赋值给了p0，但要注意的是，p是指向整型类型的，不能直接将p赋值给p0，所以这边做了类型转换，把p转换成指向字符的指针。

从运行结果可以看到，p和p0的输出都一样，但为什么*p0=1呢？

如果我们将1025用32位二进制来表示，即00000000 00000000 00000100 **00000001**。当做强制类型转换，将p赋值给了p0时，那么最右边的这个字节(即00000001)的地址会被存入p0，当解引用p0的时候，机器认为p0是一个指向字符类型的指针，字符型只有1个字节，所以机器只看1个字节。所以解引用p0(*p0)的结果就是1。

```text
//3_2.c
#include <stdio.h>

int main() {
 //1025 = 00000000 00000000 00000100 00000001
 int a = 1025;
 int *p;
 p = &a;
 printf("size of integar is %d.\n", sizeof(int));        //通过sizeof()得到int所占字节
 printf("Address = %p, value = %d.\n", p, *p);
 printf("Address = %p, value = %d.\n", p + 1, *(p + 1));  
 char *p0;
 p0 = (char *)p;      //强制类型转换，因为p指向int类型，p0指向char类型
 printf("size of char is %d.\n", sizeof(char));
 printf("Address = %p, value = %d.\n", p0, *p0);    
 printf("Address = %p, value = %d.\n", p0 + 1, *(p0 + 1)); 
}
```

![ ](https://pic4.zhimg.com/v2-35d9d15a4dc0f43040fc98fdec4cbbeb_b.jpg)

3_2.c中分别对p和p0做算术运算，并进行解引用。p+1会指向下一个整型变量的地址，但是那个地址并没有分配整型变量，所以打印出来的实际上是个随机值(垃圾值)。而p0+1会指向下一个字符型变量的地址，在程序中实际上是到了1025的第2个字节(00000100)的地址，所以*(p0+1)的值为4。

### void类型指针

还有一种指针类型是通用指针类型，它不针对某个特定的数据类型，这种指针类型被称为void类型的指针。

```text
//3_3.c
#include <stdio.h>

int main() {
 int a = 1025;
 int *p;
 p = &a;
 printf("size of integar is %d.\n", sizeof(int));        //通过sizeof()得到int所占字节
 printf("Address = %p, value = %d.\n", p, *p);

 void *p1;             //p1为通用指针类型
 p1 = p;                                 //合理
 printf("Address = %p.\n", p1);  //void类型的指针不能直接解引用 *p1会有编译错误 因为该指针没有指向任何特定的类型
// printf("Address = %p.\n", p1+1); //p1+1也是不行的

}
```

使用void来声明这种特定的指针类型，然后把星号(*)加在变量名前面。

### 本章总结

如果声明了一个指针变量p，p是指向某种数据类型的指针，那么在解引用的时候，机器只会看sizeof(data type)个字节，其中data type可以是int, float, char……

sizeof()：以字节为单位给出指定类型的大小。

## 四、指向指针的指针

![ ](https://pic3.zhimg.com/v2-d906222f734226845b0b24c3b9521ac2_b.jpg)

图4.1

假设图4.1是计算机内存的逻辑视图，这里的每一块都是一个字节，每一个字节都有一个地址。假设地址从左向右增大，并从地址200开始，201, 202……

int x=5; 声明一个整型变量x，初始化为5。程序运行的时候，会为变量x分配一些内存，假设地址225开始的四个字节分配给了x，里面存的值是5。

int *p; 声明一个指向整型的指针p，编译器会为这个p预留一些地址空间，指针也是占据4个字节(byte)，假设地址215开始的4个字节分配给了p。p是指向int类型的指针。

p=&x; 把x的地址存到p中，即p指向x。

*p=6; 通过指针p进行写数据，那么地址225开始的四个字节，里面的值就被修改成了6，这样变量x也就被修改了。

int **q; 在变量前面放两个星号(*)，声明了一个指向指针的指针q。

q=&p; q存放了p的地址，即q指向了指针p。假设q的地址是205开始的四个字节。q是指向int *类型的指针

int ***r; 声明了一个指向指针的指针的指针r。

r=&q; r存放了q的地址，即r指向了指针q。假设r的地址是230开始的四个字节。这个r是指向int **类型的指针。

![ ](https://pic3.zhimg.com/v2-593098aaab9f1468e7a6428118766a5a_b.jpg)

图4.2。图中r所存储的值应该是205(作者笔误)

图4.2中，打印*p：输出结果是6。*p是p所指向的内存地址中存储的值，所以是6.

打印*q：*q是q所指向的内存地址中存储的值，q指向的地址是215，地址215存放的值为225，因此输出的结果是225，即*q=225=p。

打印*(*q)：解引用两次。首先q=225，然后再看地址225存储的值，是6，所以输出结果是6，即**q=6。可以省略“*(*q)”中的括号，写成“**q”也是可以的。

打印**r：*r，即r所指向的内存地址中所存储的值，r指向地址205(图4.2中r所存储的值应该是205(作者笔误))，那么地址205存储的值为215，所以*r=215=q

**r，也就是*q，所以输出是225。**r=225=*q=p。

打印***r：由于**r=p，所以*(**r)=*p，而*p=6，所以***r=6，这个6也就是变量x的值。

```text
//4_1.c
#include <stdio.h>

int main() {
 int x = 5;
 int *p = &x;
 *p = 6;
 int **q = &p;
 int ***r = &q;

 printf("*p is %d.\n", *p);  //对应图4.2中的6
 printf("*q is %#x.\n", *q);  //对应图4.2中的地址225(即存储变量x的地址)
 printf("**q is %d.\n", **q);  //对应图4.2中的6
 printf("**r is %#x.\n", *r);  //对应图4.2中的地址215(即存储变量p的地址,也可以说q所存储的值)
 printf("**r is %#x.\n", **r);  //对应图4.2中的地址225(即存储变量x的地址)
 printf("***r is %d.\n", * **r);  //对应图4.2中的6

 * **r = 10;    //将地址225所存储的值修改为10
 printf("x = %d.\n", x);
 **q = *p + 2;    //将地址225所存储的值修改为12
 printf("x = %d.\n", x);
}
```

![ ](https://pic4.zhimg.com/v2-adb023c87713df5e26d5a481bf43df53_b.jpg)

### 本章总结

int *p：p为一级指针；int **q：q为二级指针；int ***r：r为三级指针…… 因此，在声明指针变量时，有n个星号(*)，就称其为n级指针。

以图4.2为例。对一级指针解引用一次，会得到变量x的值。对二级指针解引用两次，才能得到变量x的值……因此，对n级指针解引用n次，就能得到最终的变量值。

n级指针解引用1,2,3,……,n-1次的结果均是地址(n≥2且n为正整数)。

对指针的解引用，就是在获取指针所指向的内存地址中所存储的值。

## 五、函数传值vs传引用

把指针作为函数参数，被称为传引用。

先来看下面这段代码。

```text
//5_1.c
#include <stdio.h>

void Increment(int a) {
 a = a + 1;
}

int main() {
 int a;
 a = 10;
 Increment(a);
 printf("a = %d.\n", a);
}
```

![ ](https://pic3.zhimg.com/v2-0dfabacada1aefb69ce8a7ebfe87b32e_b.jpg)

5_1.c想要做的是将a自增，但是输出的结果仍然是10。原因在于main函数中的a是个局部变量，也就是只能在main函数中使用局部变量a，其他函数中无法使用该局部变量a。所以Increment中的局部变量a和main函数中的局部变量a实际上是两个a，并不是同一个a。当main函数调用Increment这个函数时，只是将main函数中的a的值拷贝到Increment函数中的局部变量a。因此输出仍然是10。

接下来在代码中打印这两个局部变量a的地址。

```text
//5_2.c
#include <stdio.h>


void Increment(int a) {
 a = a + 1;
 printf("Address of variable a in Increment = %#x.\n", &a);
}

int main() {
 int a;
 a = 10;
 Increment(a);
 printf("Address of variable a in main = %#x.\n", &a);
// printf("a = %d.\n", a);
}
```

![ ](https://pic1.zhimg.com/v2-634700bbb662824a57a0410ef3b86038_b.jpg)

5_2.c中在Increment和main函数中分别打印了局部变量a的地址，从运行结果上就能看出这两个局部变量a实际上是不一样的。

![ ](https://pic2.zhimg.com/v2-dc5752aee8364e51c499b664f126c729_b.jpg)

图5.1

当一个应用程序启动的时候，计算机会设置一些内存给这个程序使用。应用程序所使用的内存通常被分为4个部分，第一个部分(Code(Text))用来存储程序的指令，计算机需要把指令加载到内存(例如代码中的自增语句)；第二部分(Static/Global)是分配给静态或者全局变量，作为全局变量，在程序的任何地方都可以被访问和修改，不像局部变量只能在特定的函数或者特定的代码块进行访问和修改；第三块内存称为栈(Stack)，局部变量都放在这个地方；第四块就是堆(Heap)。内存中的这4个段，代码段、静态/全局变量段和栈段这三个是固定的，在程序开始运行的时候就已经确定了的。但是应用程序在运行的时候可以要求在堆区为它分配更多内存。

![ ](https://pic4.zhimg.com/v2-7d2ea020e59bd35d976c8d993268669b_b.jpg)

图5.2

图5.2中，假设程序被分配的内存是从200到800，这些是程序使用的不同的段。地址300到600是栈区。

当程序开始运行时，main函数会被调用，所有关于这个函数的信息(例如参数，局部变量……)都存在栈上，所以在栈上会开辟出一块空间给main函数，创建出的这个区域称为栈帧(Stack Frame)。每个函数都会有一个栈帧。在main函数中有一个局部变量a，那么会给a分配内存，并赋值为10。现在main函数调用Increment这个函数，机器此时会在Increment(a)这条指令停下来，去执行Increment这个函数，一旦结束之后，再次回到main函数。现在会为Increment函数开辟出另一个栈帧，并给Increment的参数a也开辟相应的内存空间，传进来的值会被拷贝到新的内存空间。当执行“a=a+1”时，这个在Increment函数中的局部变量a增大了1，它不能访问自己的栈帧之外的变量。

![ ](https://pic3.zhimg.com/v2-87eda17d6ac13ad71ef961c414e28d3a_b.jpg)

图5.3

当Increment函数调用完毕，程序控制流再一次返回到main函数，机器会清除之前Increment函数使用过的栈帧(如图5.3)，然后main函数从之前暂停的地方恢复执行。所以局部变量的生命周期在函数执行期间一直存在。

![ ](https://pic3.zhimg.com/v2-2ac4c47e44c9aa31a7a68a0896afdaaa_b.jpg)

图5.4

main函数的下一个语句是printf，printf是一个库函数，main函数的执行被暂停，转而执行printf，这种特殊的结构被称为调用栈或者函数调用栈。栈顶的函数正在执行。

![ ](https://pic2.zhimg.com/v2-1cca48b44156b20256d83750e1070ae1_b.jpg)

图5.5

在代码中，main函数为主调函数(calling function)，Increment函数为被调函数(called function)，当在主调函数中调用其他函数，这个参数就被称为实参(actual argument)，被调函数中的这个参数被称为形参(format argument)，实参会被映射到形参，当函数发生调用时，实参"a"会被映射到另外一个形参"a"。一个变量中的值拷贝到另外一个变量，这种形式的函数调用被称为传值调用。这就是为什么5_1.c输出10而不是11的原因。

那么到底如何使得main函数中的局部变量a执行自增呢？如果函数参数是指针的话就可以完成这个操作。

```text
//5_3.c
#include <stdio.h>


void Increment(int *p) {
 *p = *p + 1;
}

int main() {
 int a;
 a = 10;
 Increment(&a);
 printf("a = %d.\n", a);
}
```

![ ](https://pic3.zhimg.com/v2-498b6f98fb349506e08eda118f8d4122_b.jpg)

在5_3.c中Increment函数的参数不再是整型变量，而是一个指向整型的指针。因此在main函数中，当调用Increment函数时，传递的值是main函数中局部变量a的地址。

![ ](https://pic3.zhimg.com/v2-ff4b20ac47c36a6153d2d6b349d00552_b.jpg)

图5.6

图5.6中，假设地址300到350是main函数的栈帧，main函数中有一个局部变量a，假设局部变量a的地址是308。当main函数调用Increment函数，局部变量p会被创建出来，p是一个整型指针，p的值将会是main函数中局部变量a的地址308，所以p指向a。

"*p = *p + 1;"：*p是p指向的内存地址所存储的值，执行加1后，p指向的内存地址所存储的值会增加1，即地址308存储的值会加1，因此现在a是11。Increment函数结束之后回到main函数。继续执行printf语句，然后就会打印出11这个值。

这样的函数调用，不是传值，而是传变量的地址。这种方式可以节省很多内存空间，虽然传递变量地址也需要占用一些内存，但是相比之下会小得多，避免复杂的数据类型拷贝可以节省很多内存。

## 六、指针和数组

  

在C语言中，指针和数组的概念常常一起出现。当声明了一个整型数组A时，数组的大小是5。会创建5个整型变量，为A[0], A[1], A[2], A[3], A[4]。这五个整型数会作为一个整块，它们会被连续存储在内存中。

![ ](https://pic2.zhimg.com/v2-86dfeb705ae9ef5e4438a38a63c490c9_b.jpg)

图6.1

图6.1中，假设A[0]的地址为200，由于int类型所占字节为4byte，因此A[1]的地址是204，A[2]的地址是208，A[3]的地址是212，A[4]的地址是216，所以数组A的总大小是20byte，并且是20个字节是个连续的块。

![ ](https://pic4.zhimg.com/v2-7f8c1749815693eb2d3ecada2ee4dce7_b.jpg)

图6.2

图6.2。假设还有一个整型变量x，初始化为5，它的地址是300。还有一个指向整型的指针p，p存放了x的地址。如果直接解引用p(即*p)，可以直接得到变量x的值5。但是如果是*(p+1)的话，会得到一个随机值，因为下一个地址并没有被分配任何整型变量。

但是对于数组A来说，它的地址是200。如果声明一个整型指针p，然后把数组中第一个元素A[0]的地址赋值给p，此时直接打印p，就会得到200。打印*p，就会得到2。

如果打印p+1，那么输出204，即A[1]的地址；如果打印*(p+1)，那么输出4，即A[1]的值。以此类推，解引用p+2得到第三个元素的值，即打印*(p+2)，输出5。因此对数组使用指针算术运算就有意义了，因为此时知道相邻的地址存放了什么值。数组有个特点是，如果使用数组名A，那么会得到一个指向数组首元素的指针。所以可以这么写：

```text
//6_1.c
#include <stdio.h>

int main() {
 int A[5] = {2,4,5,8,2};
 int *p;
 p = A;     //此处可以不使用&
}
```

如果直接打印A，那么会得到数组首元素A[0]的地址200。如果对它解引用(*A)，则会得到A[0]的值2。

如果打印A+1，那么会得到A[1]的地址。如果对其解引用，*(A+1)，则会得到A[1]的值4。

对于数组中索引是i的元素，为了取得这个特定元素的地址，可以使用&A[i]或者A+i。

为了取得数组中特定元素的值，可以使用A[i]或者*(A+i)。

直接使用数组名，会得到数组的基地址，也叫数组首元素的地址。

```text
//6_2.c
#include <stdio.h>

int main() {
 int A[] = {2, 4, 5, 8, 1};
 printf("A = %#x.\n", A);
 printf("&A[0] = %#x.\n", &A[0]);
 printf("A[0] = %d.\n", A[0]);
 printf("*A = %d.\n", *A);

 return 0;
}
```

![ ](https://pic2.zhimg.com/v2-660f68cae0db4f5662e3ee2c2b983989_b.jpg)

前两行printf分别打印A和&A[0]，可以看到结果是一样的，都打印出了数组首元素A[0]的地址。

后面两行printf分别打印A[0]和*A，结果也是一样的，都打印出了A[0]的值。

```text
//6_3.c
#include <stdio.h>

int main() {
 int A[] = {2, 4, 5, 8, 1};
 int i;
        //通过for循环打印出数组A每一个元素的地址以及值
 for (i = 0; i < 5; i++) {
  printf("A = %#x.\n", A + i);
  printf("&A[0] = %#x.\n", &A[i]);
  printf("A[0] = %d.\n", A[i]);
  printf("*A = %d.\n", *(A + i));
 }

 return 0;
}
```

![ ](https://pic3.zhimg.com/v2-36d3e2fea15c87c0c6ccc2d257d7e392_b.jpg)

### 本章总结

A为数组名。

数组的特点：如果使用数组名，会得到一个指向数组首元素的指针。

&A[i]和(A + i) 都是用来获取数组中元素的地址。

A[i]和*(A + i)都是用来表示获取数组中元素的值。

代码中写A++是不可以的，因为数组名代表的是数组首元素的地址，因此A实际上是一个常数，而不是变量，因此A++是不被允许的。但是可以将A赋值给指针变量p(即p=A)，然后执行p++，这个是合理的，因为p是一个指针变量。

## 七、数组作为函数参数

```text
//7_1.c
#include <stdio.h>

int SumOfElements(int A[], int size) {   //int A[]也可以写为int* A
 int i, sum = 0;
 for (i = 0; i < size; i++) {
  sum += A[i];    //A[i]等同于*(A+i)
 }
 return sum;
}

int main() {
 int A[] = {1, 2, 3, 4, 5};
 int size;
 int total_sum;
 size = sizeof(A) / sizeof(A[0]);  //用于算出数组中有多少个元素
 total_sum = SumOfElements(A, size);  //传递数组首元素地址时，也可以写成&A[0]
 printf("total sum is %d.\n", total_sum);

 return 0;
}
```

![ ](https://pic4.zhimg.com/v2-b18f4eb7275cb4c46ebe542525115a67_b.jpg)

7_1.c是为了计算出数组中每个元素的和。代码中用数组名作为函数参数。

size = sizeof(A) / sizeof(A[0]); 是为了得出数组中有多少个元素。数组A是一个存储整型变量的数组，int类型占据4个字节，所以sizeof(A)=20=5*4byte。sizeof(A[0])为了得出数组中某一个元素所占据的字节，因此为4byte。所以sizeof(A)/sizeof(A[0])=5，由此可得数组A包含5个元素。最后循环5次去计算出数组A的元素和。

```c
//7_2.c
#include <stdio.h>

int SumOfElements(int A[]) {    //int A[]也可以写为int* A
 int i, sum = 0;
 int size;
 size = sizeof(A) / sizeof(A[0]);  //用于算出数组中有多少个元素
 for (i = 0; i < size; i++) {
  sum += A[i];    //A[i]等同于*(A+i)
 }
 return sum;
}

int main() {
 int A[] = {1, 2, 3, 4, 5};
 int size;
 int total_sum;
 total_sum = SumOfElements(A);   //传递数组首元素地址时，也可以写成&A[0]
 printf("total sum is %d.\n", total_sum);

 return 0;
}
```

![ ](https://pic1.zhimg.com/v2-e80522a2115fbcccb075050ffe961a68_b.jpg)

现在，换一种方法，不把数组的大小(size)作为额外的函数参数传进去，直接在被调函数中计算数组的大小。

但是从运行结果来看，数组的元素和输出了3，而不是15。这里试着打印一下SumOfElements函数和main函数中的sizeof(A)以及sizeof(A[0])。

```text
//7_3.c
#include <stdio.h>

int SumOfElements(int A[]) {    //int A[]也可以写为int* A
 int i, sum = 0;
 int size;
 size = sizeof(A) / sizeof(A[0]);  //用于算出数组中有多少个元素
 printf("SOE - Size of A is %d, size of A[0] is %d.\n", sizeof(A), sizeof(A[0]));   //x64系统中 指针是8Byte
 for (i = 0; i < size; i++) {
  sum += A[i];      //A[i]等同于*(A+i)
 }
 return sum;
}

int main() {
 int A[] = {1, 2, 3, 4, 5};
 int size;
 int total_sum;
 total_sum = SumOfElements(A);    //传递数组首元素地址时，也可以写成&A[0]
 printf("Main - Size of A is %d, size of A[0] is %d.\n", sizeof(A), sizeof(A[0]));   //x64系统中 指针是8Byte
 printf("total sum is %d.\n", total_sum);

 return 0;
}
```

![ ](https://pic4.zhimg.com/v2-a279b087abc099b2226343feeebb3a0f_b.jpg)

SOE表示SumOfElements函数的简称。从7_3.c的运行结果可以看出，在SumOfElements函数中sizeof(A)是8byte，而在main函数中sizeof(A)是20byte，sizeof(A[0])仍然是4byte。为什么会这样呢？来看下图7.1的分析。

![ ](https://pic3.zhimg.com/v2-54dd96b84d8a75f46d2cd58729603952_b.jpg)

图7.1

程序开始执行时，先执行main函数，那么会在栈(Stack)上开辟一个内存空间给到main函数，这块区域被称为栈帧(Stack Frame)。在这块内存空间中会分配20个字节的空间给到数组A以及4个字节的空间给到total(数组A和total都是局部变量)。当调用SumOfElements函数时，会另外开辟新的内存空间给到SumOfElements，在调用时，正常情况下main函数里面变量的值会被拷贝到SumOfElements函数中，所以理想情况是，在SumOfElements的内存空间中，会有20个字节分配给另外一个A，这个A是SumOfElements的局部变量，它和main函数中的数组A应该拥有同样的元素。但事实并不是这样，当编译器看到数组作为函数参数时，它不会拷贝整个数组，实际上编译器做的是创建了一个同名的指针，并非创建整个数组。在7_3.c中是创建了一个整型指针。编译器只是拷贝了main函数中数组A首元素的地址，假设数组A首元素的地址是200，因此在SumOfElements中仅仅是创建了一个整型指针，该指针存放地址200。

代码中声明SumOfElements函数时“int SumOfElements(int A[])”，编译器自动把int A[]转换为int *A，也就是说SumOfElements函数中的A不是被解释成了数组，而是被解释成了数组。这种方式的目的在于节省内存，因为数组万一很大的话，每次调用函数都需要拷贝整个数组就会浪费大量内存。因此对于数组来说，不使用简单的传值，而是传递数组元素的地址。这就是为什么7_3.c中SumOfElements函数中打印sizeof(A)的结果是8，而不是20，因为在SumOfElements函数中A是一个整型**指针**，在main函数中A是一个整型**数组**。(x64系统中，指针需要占据8个byte。)所以需要把数组大小也当作参数一起传递，参考7_1.c。

![ ](https://pic3.zhimg.com/v2-3efb6e5ea5c3cd2bbc1cd51ed301df42_b.jpg)

图7.2

对于指针变量来说，如果有一个指向数组首地址的指针，可以像使用数组名一样去使用它。因为A[i]会被解释为地址(A+i)的值。如图7.2的红框处。

```text
//7_4.c
#include <stdio.h>

void DoubleElements(int *p, int size) {
 int i;
 for (i = 0; i < size; i++) {
  p[i] = 2 * p[i]; //将数组的每个元素乘2
 }
}

int main() {
 int A[] = {1, 2, 3, 4, 5};
 int size, i;
 size = sizeof(A) / sizeof(A[1]);
 DoubleElements(A, size); //传递数组首元素地址时，也可以写成&A[0]，即DoubleElements(&A[0], size)
 for (i = 0; i < 5; i++) {
  printf("A[%d] = %d.\n", i, A[i]);
 }

 return 0;
}
```

![ ](https://pic3.zhimg.com/v2-e1b3b0beaab712e04307eb5448dad09e_b.jpg)

数组名作为函数参数时，可以在被调函数中修改元素的值。例如7_4.c中，定义了一个DoubleElements函数，用于将数组的每个元素都乘2，然后打印出来。通过指针的解引用，去对main函数中的数组A作修改。

### 本章总结

当数组名作为函数参数时，传递的其实是数组首元素的地址，并非整个数组。

对于指针变量来说，如果有一个指向数组首地址的指针p，可以像使用数组名A一样去使用它。p[i]等同于*(A+i)。

定义函数时，“void function(int p[])”等同于“void function(int *p)”，都是创建了一个指向整型类型的指针p。建议使用“void function(int *p)”这种写法。

## 八、指针和字符数组(上)

在C语言讨论字符数组的时候，基本上是在讨论字符串。字符串是一组字符，例如"John"、"Hello World"、 "I am feeling lucky."。

为了能够在字符数组中存储字符串，首要的需求就是字符数组必须要足够大，大到能够容纳字符串。一个足够大的字符数组，它的大小要≥字符的数量+1。比如，有一个字符串，"John"，这个字符串有4个字符，那么字符数组的大小要≥5。那么为什么需要一个额外的字符？大小为4的字符数组不就够了吗？

可以这样理解：比如现在有了一个大小为8的字符数组C，然后把字符串JOHN存放到这个数组中，按照图8.2存放。存放完之后，可以发现并没有信息来提示'N'字符串的最后一个字符，也就是说没有信息告诉我们说字符串的结尾在索引3那个位置。假如我们把字符串(存在字符数组中)传给一个函数，并且打印这个字符串，那么函数怎么知道这个字符串只到索引3就结束了呢？因为我们并没有往索引4-7写任何内容，所以那部分的内容都是垃圾值。

```text
}
```

图8.4中，声明了一个大小为6的字符数组C1，假设C1在内存这样存储(图8.4右上角)。数组在内存中是连续存储的，假设第一个字符的地址是200，一个字符占据1个字节，所以第二个字符的地址是201，再下一个是202，以此类推。

char *C2：再声明一个指向字符的指针C2，假设地址400用来存放指针变量C2。

C2=C1：将字符数组首元素地址(200)赋值给C2，此时C2就指向了字符数组首元素。直接使用数组名C1就可以表示字符数组首元素的地址，当然那也可以写成C2=&C1[0]。

可以使用这个字符指针C2，就像使用C1一样，来对字符数组进行读写。比如打印C2[1]，输出就是"O"。

比如想修改索引位置是0的字符，将其改成"A"，那么可以用"C2[0]='A'"。修改后，字符串就变成"Aello"。

C2[i]其实等同于*(C2+i)，或者说是C1[i]、*(C1+i)。

C1=C2：这个是不合理的，无意义，会产生编译错误。

C1++：同样会编译报错。我们无法对C1进行算术运算。

C2++：这个是可以的。此时字符指针C2会指向下一个元素。比如C2原本指向字符数组的第一个元素，在"C2++"后，C2指向了字符数组的第二个元素('e')。

### 本章总结

声明字符数组时，一定要确保数组大小≥字符串长度+1，这个额外的1是用于存放NULL字符的。

字符串必须是以NULL字符结尾的，即'\0'。

用双引号包围起来的内容叫字符串，用单引号包围起来的内容叫字符。

```text
eg. "JOHN" -> 字符串
    'P' -> 字符
    "p" -> 字符串
    'Hello' -> 不合理, 此时编译器会报错
```

字符占据一个字节。

初始化字符串的三种方式

```text
第一种

第二种

第三种

C[2] = 'H';

```

## 十、指针和二维数组

![ ](https://pic1.zhimg.com/v2-7b56058b41ce2835e699ac0b19fbf31c_b.jpg)

图10.1

在第六章描述了指针和一维数组的关系。如图10.1中，定义了一个大小为5的数组A，用于存储整型值。数组在内存中是连续存储的。假设数组A的首元素A[0]存储在地址200，整型占据4个字节，那么数组A的第二个元素A[1]就存储在地址204，数组的第三个元素A[2]存储在地址212，以此类推......

int *p=A：定义一个指向整型的指针p，并且将数组A首元素的地址赋值给指针p。

print A：此时会输出地址200。因为数组名A表示的是数组首元素的地址，在图9.1中该地址为200。

print *A：解引用数组首元素的地址，就能得到该地址存储的内容，也就是数组首元素的值，即2。

print *(A+2)：指针移动的实际距离取决于数据类型所占字节，int占据4个字节，所以A+2实际上是A+2*4，也就是说A+2此时是数组中第三个元素的地址，然后通过星号(*)解引用该地址，输出6

*(A+i)等同于A[i]：用来获取数组当中索引为i的元素的值。

(A+i)等同于&A[i]：用来获取数组中索引为i的元素的地址。

p=A是合理的，但是A=p是不合理的，p是一个指针变量，而数组名A是一个常量。就好比数字1就是一个常量，不可能说1这个常量还能被赋成其他值。

![ ](https://pic1.zhimg.com/v2-84e8ff235a7f6773bc4f2b52e0c1f998_b.jpg)

图10.2

现在，定义一个二维数组B，这实际上是创建了一个数组的数组。我们创建了两个一维数组(即B[0]和B[1])，每个一维数组中有3个整型元素，B[0]和B[1]每个都是包含3个整型元素的一维数组，int类型占据4个字节，那么包含三个整型元素的一维数组大小是12个字节。假设地址400开始的内存分配给B[0]，地址412开始的12个字节分配给B[1]。数组名返回数组首元素的指针，但这一次，元素并不是一个整型，而是具有3个整型的一维数组。

因此，此时如果写"int *p=B"，编译会报错。因为此时的二维数组名B返回的是一个指向一维数组(其中包含3个整型)的指针，而不是一个整型的指针。指针的类型是很重要的，不是在读地址的时候，而是在解引用的时候以及对指针进行算术运算的时候。

所以，我们需要定义一个指向一维数组的指针(其中的一维数组包含3个整型数)，即"int (*p)[3]=B"。

当打印B或者&B[0]时，输出都是400；

当打印*B或者B[0]时，此时的B[0]是一维数组名，因此打印B[0]会输出B[0]中第一个元素B[0][0]的地址，即&B[0][0]，所以输出的结果仍然是400

当打印B+1时，会输出什么呢？会是404又或是412？使用二维数组名B返回一个一维数组(其中包含3个整型)，如果对B作算术运算，像是B+1，此时就会移动到下一个一维数组(其中包含3个整型)，因此B+1实际是B+一维数组的大小，包含了3个整型的一维数组，其大小为12个字节，所以B+1输出412。当然B+1等同于&B[i]。

那么打印*(B+1)，B是一个指向一维数组的指针，B+1同样是一个指向一维数组的指针(该一维数组包含3个整型)。当解引用时，会得到整个包含了3个整型的一维数组，所以输出为412(这个412的含义是一维数组B[1]首元素的地址)。*(B+1)等同于B[1]，因此打印B[1]时，会返回一个指向B[1]的第一个元素B[1][1]的地址，即&B[1][0]。

打印*(B+1)+2，首先已经*(B+1)会返回一个指向一维数组B[1]首元素的整型指针，那么再+2实际上是在做指针的算术运算，因为*(B+1)是一个指针，它指向地址412。再+2就是将指针移动到下下个整型元素，即*(B+1)+2的实际意义是*(B+1)+2*4，从地址412开始跳过8个字节，也就是移动到地址420，因此*(B+1)+2会输出420。

*(B+1)+2也可以写成B[1]+2或者&B[1][0]+2或者&B[1][2]。

打印*(*B+1)，B返回一个指向一维数组(包含3个整型)的指针，对B解引用会得到一维数组，因此*B等同于B[0]。B[0]是一维数组名，它会返回一个指向一维数组首元素的指针，即返回一个整型指针，会得到指向地址400的一个指针。现在对这个整型指针+1，它会移动4个字节，指向下一个整型元素的地址，即地址404，所以*B+1等同于&B[0][1]。此时再看*(*B+1)最外面的星号，相当于对&B[0][1]进行解引用，那么就会得到B[0][1]的值，也就是3。*(*B+1)=3。

B[i][j] = *(B[i]+j)=*(*(B+i)+j)，这三个表达式是等价的，都是为了获取二维数组的所存储的元素值。

### 本章总结

int (*p)[3]：定义了一个指向一维数组(包含3个整型元素)的指针p。

假设定义了一个二维数组B[2][3]，这个二维数组包含了两个一维数组(即B[0]和B[1])，每个一维数组包含3个整型元素：

数组名B返回的是一个指向二维数组中第一个一维数组B[0]的指针，等同于&B[0]。那么B+1就是指向了二维数组中的下一个一维数组；

*B：返回一个指向一维数组B[0]首元素的地址。这里要注意的是B和*B虽然打印出的值一样，但是二者含义完全不一样，B表示指向**一维数组**的指针，*B表示指向一维数组**首元素**的指针。

B+n(n≥0且n为整数)：将指针指向二维数组中第n+1个一维数组B[n]。

*B+n(n≥0且n为整数)：将指针指向一维数组B[0]的第n+1个元素。

*(B+i)+j(i, j≥0且i, j为整数)：将指针指向二维数组中第i+1个一维数组B[i]的第j+1个元素。

对于二维数组，需要解引用两次，才能获取元素的值，而非地址，例如**B=B[0][0]=2。

```text
二维数组名用B来代替
B=&B[0];
*B=B[0]=&B[0][0];
B+n=&B[n]，其中n≥0且n为整数;
*(B+n)=B[n]=&B[n][0]，其中n≥0且n为整数;
B[i][j] = *(B[i]+j)=*(*(B+i)+j)，其中i, j≥0且i, j为整数.
```

## 十二、指针和动态内存 - 栈vs堆

在典型的架构中，分配给应用程序的内存可以分为四个区段，一个区段(Code)用来存放需要执行的指令；一个区段(Static/Global)存放静态或者全局变量，也就是不在函数中声明的变量，它们的生命周期贯穿整个应用程序，在应用程序运行期间都可以访问它们；一个区段称为栈(Stack)，用来存放函数调用的所有信息和所有局部变量，局部变量就是在函数内部声明的，只在函数执行期间存活。留给这3个区段的内存：代码段、静态/全局数据段、栈区，它们在运行期间的大小是不会增长的。第4种区段：堆(Heap)。

![ ](https://pic2.zhimg.com/v2-b54afcb94cc7db3cbe4ad8c170ceb909_b.jpg)

图12.1

图12.1中，绿色的方块作为栈来使用，橙色的内存预留给静态和全局变量使用。当程序运行的时候，main函数首先被调用，main函数被调用时，一些内存会被分配出来，用于main函数的执行。total是一个全局变量，所以它在全局段。栈上分配给main函数的内存也被称为main函数的栈帧，所有的局部变量，参数，函数返回地址所有这些信息都存在栈帧上。一个函数的栈帧大小是在编译期间就决定的。现在当main函数调用SqaureOfSum函数(后面简写为SOS)时，一个栈帧就会被分配给SOS函数，局部变量x,y,z都会在这个栈帧里面。现在SOS函数调用Square函数(后面简写为Sq)，就会另外一个栈帧分配给Sq函数，它也有自己的局部变量。在执行期间，任何时候都是栈顶的函数在执行，其他的函数会暂停，等待上方的函数返回一些东西然后再恢复执行。total是一个全局变量，在Static/Global段，因为该变量没有在函数内声明，我们可以在程序的任何地方访问它。

程序开始执行的时候，操作系统分配了一些内存，假设OS分配了1MB的内存用来作为栈，但实际的栈帧分配和局部变量的分配是在运行时。如果栈增长超出了预留的内存大小，比如A调用B，B调用C，C调用D……以此类推最终耗尽了栈空间，这被称为栈溢出(Stack Overflow)，这种情况会造成程序崩溃。一种导致栈溢出的情况是写了有问题的递归函数导致无穷递归。内存中预留给栈的空间在运行期间不会增长，应用程序不能在运行期间请求更多栈。假设预留的是1MB，那么当分配给变量和函数的栈大小超过1MB时，程序就会崩溃。另外，内存在栈上的分配和销毁有一定的规则，当一个函数被调用时，它被压入堆栈，当它结束时，弹出堆栈。如果变量是在栈上分配的，那么就不能操作变量的范围。另外一个限制是，如果需要声明一个很大的数据类型，比如一个很大的数组作为局部变量，我们需要在编译期间就知道数组的大小。如果有这么一个场景，需要在运行期间根据参数决定数组大小，那么使用栈就有问题了，针对这些问题，比如分配很大的内存或者把变量预留在内存中直到我们想用的时候为止，因此就有了堆(Heap)。

不同于栈，应用程序的堆是不固定的，它的大小在应用程序的整个生命周期是可变的，也没有特定的规则来分配和销毁相应的内存，程序员完全可以控制在堆上分配多少内存，数据保留到什么时候，几乎可以任意使用堆上的内存，只要不超出系统自身的内存限制。

堆也被称为动态内存，使用堆内存意味着动态内存分配。

为了在C中使用动态内存，需要知道4个函数，malloc, calloc, realloc, free。

![ ](https://pic3.zhimg.com/v2-59cb7cabf8285fb325be0d6e3dc078f2_b.jpg)

图12.7

"p = (int *)malloc(sizeof(int));"：通过malloc函数请求在堆上分配多少字节的内存。当调用malloc并且传入值是一个整型的大小，那么相当于在堆上请求开辟一块内存，大小是4个字节(int占据4个字节)。这样一来，malloc会在堆上申请和分配4个字节的内存，返回一个指向这块内存起始地址的指针。假设返回的4字节的起始地址为200，那么malloc将会返回200。现在就有了一个整型指针p，它被分配在main函数的栈帧上，它存储着地址200。malloc返回的是一个void类型的指针，而p是一个整型指针，所以做了强制类型转换。

"*p = 10;"：通过解引用p，往地址200写入10这个值。使用堆上内存的唯一方式，就是通过解引用。malloc函数所做的事情仅仅只是从堆上找到空闲的内存，然后返回一个指针。

再次调用malloc，"p = (int *)malloc(sizeof(int));"，堆上会再次分配4个字节，假设这次的内存从地址400开始，将地址400赋值给p，那么现在p指向的地址是400；在通过"*p = 20;"往地址400写入20这个值。

![ ](https://pic1.zhimg.com/v2-48fb81d605ab576b72689ca5206c6ab8_b.jpg)

图12.8

之前开辟的堆上内存地址200仍然有效，仍然在消耗堆内存，它不会被自动回收。当使用malloc在堆上动态分配了一些内存，那在使用完成之后需要释放它，否则就会浪费内存(即内存泄漏)。因此当我们使用完堆上的地址200后，应该调用free。使用malloc分配的内存，最终通过调用free进行释放，即"free(p)"，这样一来，地址200就会被释放，如图12.8。

![ ](https://pic4.zhimg.com/v2-d238f10121de1e42946ed04b018de4bf_b.jpg)

图12.9

如果想要在堆上创建一个数组，只需要调用malloc传入总的数组大小字节数。比如想要在堆上创建大小为20的一维整型数组，那么可以通过"p = (int *)malloc(20*sizeof(int));"，然后malloc在堆上会分配一个连续的并且足够存放20个整型的内存。malloc会返回基地址，假设基地址为50，将该地址赋值给p，基地址实际就是一维数组首元素的地址。接着可以通过"p[0]=1;p[1]=2;"给数组赋值。

如果malloc找不到空闲的内存块，也就是说不能在堆上成功分配内存，此时会返回NULL。为了处理这种错误，我们需要知道这个情况，并在代码中作出相应处理。

## 本章总结

malloc返回的是一个void类型的指针。

使用堆上内存的唯一方式，就是通过解引用。或者说访问堆上内存的方式只能通过解引用。

任何在堆上分配的内存，在函数调用结束之后并不会像栈那样自动释放。如果分配了内存，之后不再需要使用，程序员有义务释放堆上不再使用的内存，避免造成内存泄漏。

在C中使用动态内存，需要知道4个函数，malloc, calloc, realloc, free。

当定义了一个较大的数组时，我们可以将数组存放到堆上，避免存放到栈上影响栈内存空间。

## 十三、指针和动态内存 - malloc calloc realloc free

这一章会通过实际代码演示堆上分配内存的相关函数malloc、calloc、realloc以及在堆上释放内存的函数free的用法。

![ ](https://pic1.zhimg.com/v2-b286ae7ebde0858ae91f67cc0c6d553c_b.jpg)

图13.1

malloc函数定义：void* malloc(size_t size) ，参数是内存块的字节大小，注意这个数据类型为size_t，它是一个正整数，可以把它想象成unsigned int类型，即大小不能是负数，可以是0或者一个正数。malloc返回一个void指针，这个指针指向了分配给我们的内存块的第一个字节的地址。

malloc返回的是一个void类型指针，所以必须要强制转换。即"int *p = (int *)malloc(sizeof(int))"

可以通过"*p=2;"、"*(p+1)=4"给堆上的内存地址赋值或者用"p[0]=2;"、"p[1]=4"给堆上的内存地址赋值。

图13.2

calloc函数定义：void* calloc(size_t num, size_t size)，calloc接受两个参数，第一个参数num是特定类型元素的数量，第二个参数size是类型的大小。因此"malloc(3*sizeof(int))"可以写成"calloc(3, sizeof(int))"。calloc也是返回一个void类型指针。calloc和malloc有一个区别在于malloc分配完内存之后并不对对其初始化，因此如果没有填入值，将会得到一些随机值；但是用calloc的话，会对其初始化为0。

![ ](https://pic3.zhimg.com/v2-5043b05ba2f6098d3d948ffdf99914ba_b.jpg)

图13.3

realloc函数定义：void* realloc(void* ptr, size_t size)，第一个参数ptr是指向已分配内存的起始地址的指针，第二个参数是新的内存块的大小。如果有一块动态分配的内存，然后想要修改内存块的大小，那么就可以使用realloc。

如果我们想要的新内存块比原来的内存要大，这种情况下机器可能会创建一块新的内存然后把原来的值拷贝过去。如果之前的那块内存(的相邻部分)还有连续的内存可用，那么可能会直接扩展之前的那块内存。

```c
#include <stdlib.h>

int main() {
 int n;
 printf("Enter size of array.\n");
 scanf("%d", &n);
 int *A = (int *)malloc(n * sizeof(int));
 for (int i = 0; i < n; i++) {
  A[i] = i + 1;    //给动态内存赋值
 }
 for (int i = 0; i < n; i++) {
  printf("A[%d] = %d.\n", i, A[i]); //打印数组
 }

 return 0;
}
```

```text
//13_3.c

#include <stdlib.h>

int main() {

 for (int i = 0; i < n; i++) {
  A[i] = i + 1;    //给动态内存赋值
 }
 free(A);                                        //释放内存
 for (int i = 0; i < n; i++) {
  printf("A[%d] = %d.\n", i, A[i]); //打印数组
 }

 return 0;
}
```

![ ](https://pic4.zhimg.com/v2-4300a6207783d398af2c9f8c8a94c767_b.jpg)

任何分配了的动态内存在程序结束之前会一直存在(占据内存)，除非程序员显式地释放它。函数malloc, calloc, realloc分配的内存，要使用函数free来释放。函数free的入参是内存的起始地址。

13_3.c中"free(A)"会发生什么呢。如果free(A)，内存中的数据可能会被清除也可能不会被清除，这取决于编译器。Dev C++在free之后会打印出随机值。free之后的那块内存就可以被另一个malloc来分配使用了。显而易见的是尽管释放了内存，但是在打印的时候仍然可以访问那块内存，"printf("A[%d] = %d.\n", i, A[i])"仍然可以得到那块内存的值A[i]，这是使用指针的时候一个危险地方。如果我们知道地址，我们可以看那个地址中存放的值，但是应该只去读写分配给我们的内存。如果那块内存不是分配给我们的，那么我们将不知道读写的地址上会是什么。

```c
//13_4.c
#include <stdio.h>
#include <stdlib.h>

int main() {
 int n;
 printf("Enter size of array.\n");
 scanf("%d", &n);
 int *A = (int *)malloc(n * sizeof(int));
 for (int i = 0; i < n; i++) {
  A[i] = i + 1;    //给动态内存赋值
 }
 int *B = (int *)realloc(A, 2 * n * sizeof(int));
 printf("Previous block address = %#x, new address = %#x.\n", A, B);
 for (int i = 0; i < 2 * n; i++) {
  printf("B[%d] = %d.\n", i, B[i]); //打印数组
 }

 return 0;
}
```

## 十四、指针和动态内存 - 内存泄漏

内存泄漏指的是我们动态申请了内存，但是即使是使用完了之后也从来不去释放它，看起来我们的应用程序只是占据了一些未使用的内存，为什么会称之为内存泄漏呢？为什么它只会由不正确的动态内存(堆)的使用引起，而不是由应用程序内存中的其他区段引起的呢？



14_1.c中Play函数的字符数组C会分配在栈上，现在在14_2.c中想将Play函数的局部变量分配在堆上，声明一个字符指针C，使用malloc分配3个字符变量的内存，然后给内存地址赋值，这样我们就在堆上创建好了字符数组。C仍然是个局部变量，但现在C是一个字符指针，仍然可以像使用数组那样去使用它。

14_2.c的运行结果和14_1.c的运行结果并无差异，但是随着用户玩的次数越来越多，exe可执行文件的大小会越来越大。为什么当C是字符数组的时候，也就是在栈上的时候内存没有增长。

![ ](https://pic4.zhimg.com/v2-a5983fdecc8a1dcb9e73ccad5e364997_b.jpg)

图14.2

首先第一种情况，当C是字符数组的时候：main函数每调用一次Paly函数，在栈上就会为Play函数开辟一个栈帧，字符数组C也会保存在这个栈帧中，当Play函数调用结束，栈帧会自动释放，连同字符数组C也会被释放。因此栈上的空间释放不需要程序员担心，栈会自己主动释放。

  

第二种情况，使用malloc在堆上分配数组：当调用Play函数时，此时不会再栈帧上创建字符数组了，只有一个局部变量的字符指针C存放在栈帧中，这个字符指针会指向堆中的字符数组。当Play函数调用结束，其栈帧就会被释放，所有的局部变量都消失了，但是堆上的字符数组此时并未被清除，它没被释放，堆上的内存必须显式地通过free来释放。当用户玩的次数越多，那么在堆上创建的字符数组就会越多，每个字符数组都在堆中占据了3个字节，导致内存消耗随次数增长。堆不是固定大小的，只要不超出系统自身限制，应用程序就可以申请更多的内存，此时如果不释放不再使用的堆内存，就会浪费内存空间。

### 本章总结

内存泄漏总是因为堆中未使用和未引用的内存块才发生的。所以程序员有义务通过free函数及时释放堆中不使用的内存块

栈上的内存是自动回收的，栈的大小是固定的，最多就是发生栈溢出。

指针是另外一种数据类型，指针存储了另外一个数据的地址，因此函数返回指针也是完全允许的


来看15_1.c的代码，这段代码本意是想计算变量a,b的和，但是代码中有一个逻辑错误。

如图15.1，当程序开始执行的时候，会先为main函数在栈上开辟一个栈帧，假设地址是100到130，里面存放了三个局部变量a,b,ptr，假设a的地址是100，b的地址是112，ptr的地址是120。当调用Add函数时，main函数暂停执行，栈上分配新的内存给到Add函数，任何时候正在执行的函数是栈顶的那个函数。假设Add函数的栈帧地址是从130到160，Add也有3个局部变量a,b,c，这里a和b都是整型指针，c用来计算*a+*b的值，结果是6。

```text
 *c = (*a) + (*b);
 return c;
}

int main() {
 int a = 2, b = 4;
 int *ptr = Add(&a, &b);
 printf("Sum = %d.\n", *ptr);


 return 0;
}
```

### 本章总结

从栈底向上传一个局部变量或者一个局部变量的地址是可以的。

从栈顶向下传一个局部变量或者一个局部变量的地址是不可以的。

## 十六、函数指针

函数指针：用来存储函数的地址。在前面的章节中，我们使用指针主要用来存放其他变量的地址。

指针用于指向或者引用内存中的数据，这里的数据未必一定是变量，也可以是常量。也可以用指针来存储函数的地址，换句话说，我们可以定义一个指向函数的指针。我们可以用这样的指针来解引用和执行函数。

当我们说函数指针存放函数地址的时候，是在说函数指针存放了函数在内存中的起始地址或者说入口点。

```c
//16_1.c
#include <stdio.h>

int Add(int a, int b) {
 return a + b;
}

int main() {
 int c;
 int (*p)(int, int);        //必须要有括号
 p = &Add;                  //也可以写成p = Add;                 
 c = (*p)(2, 3);            //也可以写成c = p(2,3);
 printf("c = %d.\n", c);

 return 0;
}
```

![ ](https://pic4.zhimg.com/v2-995a50da43ace116a15d924754ecec6f_b.jpg)

16_1.c通过函数指针p来计算2+3的值

"c = (*p)(2,3);"：首先使用星号*解引用来获得这个函数，然后传入了两个参数2和3。也可以直接写成"c = p(2,3);"，所以当我们使用函数指针名时就像是在使用函数名一样。

```text
#include <stdio.h>
```

```text
 return a + b;
}

int main() {
 void (*p)(int, int);  //函数指针p的返回类型是void
 p = &Add;   //不合理。函数指针p的返回类型是void，Add函数的返回类型是int

}
```

```text
#include <stdio.h>

 return a + b;
}

int main() {
 int (*p)(int);  //函数指针p的有一个参数
 p = &Add;  //不合理。函数指针p的有一个参数，Add函数的有两个参数

 return 0;
}
```

需要注意的是，声明函数指针的时候，必须带有括号，如果是下面这样，则是声明了一个函数p，它返回一个整型指针。

```c
int *p(int, int);        //声明了一个函数p，它返回一个整型指针
```

## 十七、函数指针的使用案例(回调函数)

函数指针可以被用来作为函数参数，接收函数指针的这个函数可以回调函数指针所指向的那个函数。

```text

void A() {
 printf("Hello");
}

void B(void (*ptr)()) {

}

int main() {
 void (*p)() = A;
 B(p);


 return 0;
}
```

![ ](https://pic1.zhimg.com/v2-810724d5642f3d89b50c1e9bc9fab41c_b.jpg)

17_1.c中，B函数通过函数指针ptr调用它所指向的函数，换句话说，我要回调传给我的那个函数，也就是通过指针传给我的那个函数。

main函数声明了一个函数指针p，不带返回类型和参数，将函数指针p初始化为函数A的地址，然后调用B，向它传入指针p，输出了"Hello"。函数A是通过回调的方式被执行的。

main函数中的代码也可以简写为：

```text
int main() {
 B(A);

 return 0;
}
```

一个函数的引用传给另外一个函数的时候，那个函数被称作回调函数。因此函数A为回调函数。

```text
//17_2.c
#include <stdio.h>

void BubbleSort(int *A, int n) {
 int i, j, temp;
 for (i = 0; i < n; i++)
  for (j = 0; j < n - 1; j++) {
   if (A[j] > A[j + 1]) {
    temp = A[j];
    A[j] = A[j + 1];
    A[j + 1] = temp;
   }
  }
}

int main() {
 int i;
 int A[] = {3, 2, 1, 5, 6, 4};
 BubbleSort(A, 6);
 for (i = 0; i < 6; i++)
  printf("%d ", A[i]);

 return 0;
}
```

![ ](https://pic4.zhimg.com/v2-9c9c1cc26c859fbab9d1a7953e4cbb3b_b.jpg)

现在有这么一个需求，我希望在程序中，有时候我想要升序排列，有时候想要降序排列。怎么做呢？其中一个方法是写两个函数，一个用于升序，另外一个用于降序，但这样会有很多的冗余重复代码。换个角度，我们可以多传一个参数给排序函数，通过这个参数表明想要升序还是降序。

```text
//17_3.c
#include <stdio.h>

int compare(int a,int b)
{
 if(a>b) return 1;
 else return -1;
}


void BubbleSort(int *A, int n, int (*compare)(int, int)) {
 int i, j, temp;
 for (i = 0; i < n; i++)
  for (j = 0; j < n - 1; j++) {
   if (compare(A[j], A[j+1]) > 0) {
    temp = A[j];
    A[j] = A[j + 1];
    A[j + 1] = temp;
   }
  }
}

int main() {
 int i;
 int A[] = {3, 2, 1, 5, 6, 4};
 BubbleSort(A, 6, compare);
 for (i = 0; i < 6; i++)
  printf("%d ", A[i]);

 return 0;
}
```
